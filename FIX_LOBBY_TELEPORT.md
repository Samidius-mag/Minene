# Исправление проблемы телепортации в лобби

## Проблема
Игрок телепортируется к правителю Омеги (координаты ~0.5, 100, 0.5) вместо лобби (координаты 0.5, 200, 0.5).

## Причины

1. **Старый конфиг на сервере** - файл `plugins/MineneLobby/config.yml` может содержать старые координаты
2. **Лобби не создано** - если лобби не было создано, игрок может попасть в точку спавна
3. **Проблема с телепортацией** - метод телепортации может не работать правильно

## Решение

### Шаг 1: Проверьте конфиг на сервере

На сервере проверьте файл `plugins/MineneLobby/config.yml`:

```bash
cat plugins/MineneLobby/config.yml | grep -A 5 "lobby:"
```

Должно быть:
```yaml
lobby:
  world: world
  x: 0.5
  y: 200
  z: 0.5
```

**НЕ должно быть:**
```yaml
lobby:
  y: 100  # ❌ Это старые координаты!
```

### Шаг 2: Обновите конфиг

Если конфиг старый, обновите его:

1. Остановите сервер
2. Удалите старый конфиг: `rm plugins/MineneLobby/config.yml`
3. Запустите сервер - конфиг создастся автоматически с правильными координатами
4. Или скопируйте новый конфиг из проекта

### Шаг 3: Пересоберите и установите плагин

```bash
# Пересоберите плагин
cd plugins/MineneLobby
mvn clean package
cd ../..

# Установите на сервер
./install-plugins.sh [путь_к_серверу]
```

### Шаг 4: Проверьте логи

После перезапуска сервера проверьте логи:

```bash
tail -f logs/latest.log | grep -i lobby
```

Должны быть сообщения:
- `Лобби создано!`
- `Точка спавна мира установлена на лобби: Location{world=...,x=0.5,y=200.0,z=0.5...}`
- При входе игрока: `Игрок ... телепортирован в лобби на координаты: Location{...,y=200.0,...}`

### Шаг 5: Проверьте координаты в игре

После входа в игру нажмите F3 и проверьте координаты:
- **Должно быть:** y ≈ 200 (лобби)
- **НЕ должно быть:** y ≈ 100 (правитель)

## Быстрое исправление

Если проблема сохраняется:

1. Удалите конфиг лобби: `rm plugins/MineneLobby/config.yml`
2. Перезапустите сервер
3. Проверьте, что создался новый конфиг с y: 200
4. Если нет - скопируйте `plugins/MineneLobby/src/main/resources/config.yml` в `plugins/MineneLobby/config.yml`

## Проверка координат лобби

В игре используйте команду:
```
/tp 0.5 200 0.5
```

Если вы окажетесь в белой комнате из шерсти - это лобби. Если нет - лобби не создано или координаты неправильные.
